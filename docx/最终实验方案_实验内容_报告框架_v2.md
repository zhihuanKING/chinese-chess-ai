# 中国象棋博弈 AI 课程设计 · 最终实验方案 / 实验内容 / 报告框架（v2 整合定稿）

> 本文整合：原综合方案 + 一周可落地优化 + 真实硬件实测 + 2022–2026 前沿检索 + 网络规模双轨决策。**作为执行定稿，取代之前分散的几份。**
> 三句话主线：**让一个深度神经网络通过"自己跟自己下棋"从零学会中国象棋(全程 AI 互搏，不涉及人机)；用监督冷启动 + Gumbel 低模拟自对弈在一周内破局；并用实验证明它能赢过传统 Alpha-Beta 引擎，量化"算力/样本—棋力"的权衡。**

## 2. 技术栈与语言分工

| 模块 | 语言 | 说明 |
|---|---|---|
| 规则内核(棋盘/走法/将军飞将/重复和棋/zobrist/编码) | **C++17** | perft 硬验证；**pybind11** 暴露给 Python；静态库冻结复用 |
| 向量化批量 MCTS / 自对弈 | **Python + PyTorch 2.x(cu124)** | 参考 **TurboZero / MCTX**，不从零写 |

| 评估/Elo/绘图 | Python(BayesElo/Ordo, matplotlib) | — |

> 关键简化：**推理放在每个 self-play worker 进程内(纯 PyTorch)，不上 LibTorch / 自定义 C++ 推理服务**——一周可落地的最大风险消除点。

## 3. 算法路线（核心 + 2022–2026 升级）

```
核心范式:   AlphaZero (策略-价值网络 + MCTS + 自对弈)
核心算法:   Gumbel-AlphaZero (ICLR 2022)  —— 低模拟数(16~32)仍保证策略改进
先行/兜底:  标准 PUCT-MCTS 先跑通学习曲线，Gumbel 作同接口可插拔升级
样本效率+:  ReZero/Reanalyze 思路(2024) —— 用 buffer 旧数据重算 MCTS 目标
省算力+:    KataGo 四件套 —— playout cap randomization / forced playouts /
            auxiliary policy targets / global pooling (便宜高 ROI)
主题呼应:   引用 Search-contempt(2025) —— 有限算力训 AlphaZero 类引擎
冷启动:     监督预训练(AlphaGo 两阶段思想) + Warm-Start 增强
调研不实现: MuZero(规则已知不必学模型) / DeepCFR·DeepStack·Libratus·Pluribus
            (不完全信息，象棋不适用) / Student of Games·AlphaZe**(统一框架，辨析)
            / Searchless Chess·Chessformer·Maia(Transformer 路线，展望)
参考实现:   LightZero(NeurIPS23) / OpenSpiel —— 对拍降风险
```

## 4. 网络规模：双轨（用空闲卡并行）

> 瓶颈是**自对弈吞吐**不是显存；网络越大每步评估越慢→数据越少→可能"吃不饱"。**但有 Pikafish 监督数据打底，敢用中网做最终主力。**

| 轨 | 规模 | 参数 | 角色 |
|---|---|---|---|
| 小网 | **128ch×10** | ~2–3M | 跑通管线、快迭代、消融基线 |
| **中网(最终主力)** | **192–256ch×15** | ~10–20M | 最终最强权重 + 主对照 |
| 大网(上限) | 256ch×20 | ~25M | 仅作规模消融上限(国际象棋 AlphaZero 也就 20 块) |

**独享 8 卡 = 一个实验集群，两种模式切换(关键理念：不是"一个任务用8卡"，而是"既跑最强模型又并行跑消融")**

> 瓶颈在自对弈产数据，**actor 卡数 >> learner 卡数**；小网在 40GB 卡上单 worker 只用个位数 %，**每卡必须塞 2–4 worker(或 CUDA MPS)、并行对局拉到 1000+、batch 2048+**，否则浪费卡。

**模式 A — 旗舰冲棋力(D3–D5，先拿主 Elo 曲线+最强权重)**
```
GPU0-1  learner DDP   256×15 主网, BF16, batch 4096  (NVLink)
GPU2-7  self-play      6 卡 ×(2~3 worker/卡)=12~18 worker
                       每 worker 512~1024 并行对局, 大 batch 推理吃满卡
ReplayBuffer /dev/shm 200万~500万；数据/权重落 /mnt/nvme3n1
CPU 192线程  走法生成 + 同时并行 Arena 评估对局(不占 GPU)
```
**模式 B — 消融舰队(D5–D6，并行而非串行，一夜出齐)**
```
8 卡切 6~8 个独立训练任务同时跑(每个 1 卡自对弈+训练，小网)：
  n_sim=16 / 32 / 64 / 200  ·  冷启动off  ·  网络=256×20  ...
```
> 也可全程混合：**4 卡旗舰 + 4 卡消融舰队**，GPU 全天接近满载。单任务效率不如 C++ 集中推理服务，用"每卡多 worker + 并行多实验"的集群级聚合利用率补回，且不碰 C++ 深坑。
