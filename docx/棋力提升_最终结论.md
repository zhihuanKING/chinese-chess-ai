# 中国象棋 AI 棋力提升——最终结论（2026-06-15）

> 目标：纯粹提升神经网络的**绝对棋力**（外部引擎锚点，非 net-vs-net）。时间预算一周。
> 唯一度量：对 random / 自研 Alpha-Beta(200・500ms) / Pikafish(d1-d4) 的胜率与 Elo，修复版协议（引擎收完整着法历史），≥64 局，关键结论 128 局。

## 一、最终成果：r1 模型，引擎锚点平均 +203 Elo

交付权重 `deliverable/r1_champion_256x15.pt`（256×15，v1 监督底座 + DAgger 分布修复蒸馏）。
128 局/锚点权威终测（vs v1 起点对比）：

| 锚点 | r1 胜率 | r1 Elo | v1→r1 ΔElo |
|---|---|---|---|
| random | 100% | (饱和) | — |
| 自研 AB-200ms | 27.3% | −170 | **+144** |
| 自研 AB-500ms | 28.5% | −160 | **+251** |
| Pikafish d1 | 20.3% | −237 | **+173** |
| Pikafish d2 | 7.8% | −429 | +94 |
| Pikafish d3 | 9.4% | −394 | +204 |

**六引擎锚点平均 +203 Elo。** 对自研 Alpha-Beta 从 ~10% 提到 ~28%、对 Pikafish d1 从 8.6% 提到 20.3%。未达"战胜 Alpha-Beta"（需 >50%），但把差距收窄了约 200 Elo。

## 二、有效的与无效的（全部外部锚点证据）

**✅ 有效（贡献全部增益）——监督蒸馏 + 分布修复（DAgger）**
- v1 监督模型是最强底座（三底座实测：v1 −516 ≫ safe −693 ≫ v2cold −815 平均外部 Elo）。
- round-1 关键一步：把训练局面从"随机走子开局"换成"模型自己的实战分布 + 对引擎败局"，用 Pikafish depth-14 标注，从 v1 续训。**这一步 = 全部 +203 Elo 的来源。**机理：修复了训练分布与实战分布的系统性错配。

**❌ 无效（已严格证伪，不要再试）**
1. **标签公式工程**（A0）：WDL 校准锐化 value **有害**（浅层 MCTS 均值回传里饱和失区分度，旧 tanh/500 的平缓反而适配低模拟搜索）；policy 温度调尖无增益。
2. **自对弈 RL**（B 循环）：三配方全部无法超过 r1 外部基线——lr1e-3 头对头崩溃、lr3e-4 外部单调跌、冻value(safe配方)跑 14h/63832步/79候选/**0 晋升**。机理：自对弈改进算子（n_sim=128）在已较强的蒸馏底座上产不出更好的训练目标。**注**：此前 safe 的 +122 是在**弱**底座(v2cold ~0.10)上有爬升空间才成立；底座一强（r1 ~0.20）即失效。

## 三、方法论副产品（比棋力数字更可复用）

1. **net-vs-net Elo 不可信**：相对参考网 +301 可对应外部 ≈0（通胀）；候选打不过自己的蒸馏老师却外部不弱（通缩）。**任何"变强了吗"只认外部锚点。**
2. **评测协议是隐形地雷**：引擎不发着法历史→对重复失明→旧基线虚高 ~200 Elo 全是泡沫。先修尺子再谈提升。
3. **门控三连坑**（均已修）：赢者诅咒（中奖测量被当基线棘轮抬高门槛）、评测-晋升竞态（晋升的权重≠被评测的权重）、torch 抢满 CPU 饿死自对弈 worker。
4. **棋力天花板的本质**：我们每步 200 模拟 vs 引擎每步几十万节点，Elo 指数尺度下同预算极不对称；蒸馏上限 = 老师在标注深度的判断的一次前向压缩，丢失决定胜负的战术尾部。

## 四、若继续推（未做，留给后续）

- **容量杠杆**（最值得试）：r1=256×15，train loss 平台 2.52 可能是容量限。训更大网(384×20)吃同样蒸馏数据，测容量是否是瓶颈。需多卡 ~3h。
- round-2 DAgger 迭代：数据仅多 9%，EV 递减。
- 提高自对弈 n_sim（256+）让改进算子更强——但 B 循环已证当前底座下 RL 收益微弱，性价比低。
